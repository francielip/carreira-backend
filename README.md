# Pensamento computacional: fundamentos da computação e lógica de programação
01. Mundo da Computação

Computar significa processar alguma informação.
Dados de entrada > processamento > dados de saída.
Operações lógicas: exemplo do ar condicionado, termos uma decisão sobre estar acima ou abaixo da temperatura ideal.
Operações aritméticas: exemplo de somar valores, diminuir valores...
Hardware: todos os componentes físicos do computador, como placas, circuitos eletrônicos, teclado, mouse e tela.
Software: desenvolvido por pessoas desenvolvedoras, que atuam com diferentes especialidades em equipes multidisciplinares para construir a solução. 
Um software não é uma peça única. Ele é formado por diferentes componentes, por isso precisamos de equipes multidisciplinares, com pessoas de diferentes especializações e experiências. Em geral, designamos como componentes básicos de um software o back-end, o front-end e o banco de dados.
O back-end concentra as operações lógico-aritméticas da aplicação. Toda a parte de processamento de dados ocorre no back-end, onde construímos e passamos instruções para o computador realizar as operações pertinentes à aplicação.
O front-end é onde as pessoas usuárias interagem com a aplicação. É a face mais visível do software.
O banco de dados é o componente onde armazenamos todos os dados gerados a partir da interação com a pessoa usuária.
Podemos entender os softwares como sequências organizadas de decisões e ações, baseadas nas instruções escritas pelas pessoas desenvolvedoras. Utilizamos o pensamento computacional para organizar essas sequências de instruções.
Todas as funcionalidades que precisam estar dentro da nossa aplicação nascem a partir do entendimento do problema que queremos resolver. Por isso, o primeiro passo dessa abordagem é compreender o problema.


Atividades:

01
A Organo, uma plataforma para criação e gestão de organogramas empresariais, está desenvolvendo uma nova funcionalidade que permite às pessoas usuárias criar organogramas dinâmicos que se ajustam automaticamente a mudanças na estrutura organizacional, como promoções ou transferências de funcionários. A equipe de desenvolvimento que você faz parte precisa estruturar os dados de forma que essas mudanças sejam refletidas instantaneamente no organograma.

Qual abordagem seria mais eficiente para estruturar esses dados?

Resposta: Decompor o problema em partes menores, identificar padrões nas mudanças organizacionais, abstrair informações relevantes e criar um modelo de dados que permita atualizações rápidas e eficientes, garantindo que o organograma reflita instantaneamente as mudanças.

Comentário: Essa abordagem utiliza o pensamento computacional de forma abrangente, decompondo o problema, reconhecendo padrões, abstraindo informações e criando um modelo de dados eficiente, o que é essencial para a atualização dinâmica do organograma.

02
Uma loja online de roupas e acessórios está desenvolvendo um aplicativo com realidade aumentada para que os clientes visualizem como ficariam usando os produtos antes da compra.

Qual seria a melhor abordagem para integrar as diferentes especialidades da equipe (como desenvolvimento, design, marketing e atendimento ao cliente) e garantir o sucesso do projeto?

Resposta: Promover uma colaboração eficaz entre as equipes de design, desenvolvimento de software e marketing, com reuniões regulares e comunicação clara, além de realizar testes de usabilidade para garantir uma experiência intuitiva e envolvente.

Comentário: Essa abordagem assegura que todas as equipes trabalhem em conjunto para criar um aplicativo de realidade aumentada que seja funcional, atraente e bem comunicado aos clientes, alinhando objetivos e resolvendo problemas rapidamente.
